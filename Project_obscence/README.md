# Анализ файла с комментариями на наличие нецензурных выражений

Этот проект содержит скрипты для анализа CSV-файла с комментариями и добавления колонки, указывающей на наличие мата или нецензурных выражений.

## Файлы

- `obscene.csv` — исходный файл с комментариями (в формате: каждая запись состоит из двух строк: строка с комментарием, начинающаяся с кавычки, и строка с одной кавычкой-разделителем).
- `obscene_with_flag.csv` — результат обработки моделью GPT5: CSV-файл с двумя колонками: `comment` (текст комментария) и `obscene` (значение `yes`/`no`).
- `obscene_with_flag_old.csv` — результат обработки моделью DeepSeek.
- `obscene_with_flag_AI.csv` — результат обработки AI агентом с использованием ML модели.
- `add_column.py` — скрипт, сгенерированный моделью DeepSeek.
- `add_column_modify.py` — скрипт, сгенерированный моделью GPT5.
- `add_column_modify with_AI.py` — скрипт с использованием AI агента.
- `check_output.py` — скрипт для проверки результатов и получения статистики, а также сравнения трех моделей.
- `README.md` — эта инструкция.

## Использование

### Требования
- Python 3.6+
- Стандартные библиотеки (re, csv)

### Запуск
1. Поместите файл `obscene.csv` в ту же директорию, что и скрипт.
2. Выполните команду:
   ```bash
   py add_column.py
   ```
   или
   ```bash
   python add_column.py
   ```
3. После выполнения будет создан файл `obscene_with_flag.csv` с результатами.

### Описание работы скрипта
Скрипт выполняет следующие шаги:
1. Читает исходный файл в кодировке UTF-8 с BOM (`utf-8-sig`).
2. Пропускает строки, состоящие только из кавычки (`"`).
3. Извлекает текст комментариев, удаляя обрамляющие кавычки.
4. Проверяет каждый комментарий на наличие нецензурных слов с помощью регулярных выражений.
5. Записывает результат в новый CSV-файл с колонками `comment` и `obscene`.

### Правила определения нецензурных выражений
Используется список регулярных выражений, охватывающих основные русские матерные корни и оскорбительные слова:
- хуй, пизд, еб, бля, сук, муд, гандон, пидор, дебил, долбоёб, урод, шлюха, говн, дерьм, залуп, чмо, срак, сос, выеб, перееб, дроч, манда, хер, хрен, сучк, жоп, анус и др.
- Также учитываются некоторые оскорбительные выражения (тупой, идиот, даун, кретин).

Скрипт не претендует на 100% точность, возможны как ложные срабатывания, так и пропуски.

### Результат
Выходной файл содержит:
- Заголовок: `comment,obscene`
- Для каждого комментария: текст комментария (экранированный при необходимости) и метка `yes` (есть мат) или `no` (нет мата).

Пример первых строк:
```
comment,obscene
"Верблюдов-то за что? Дебилы, бл...",yes
"Хохлы, это отдушина затюканого россиянина...",no
```

### Статистика
После обработки можно получить статистику, запустив скрипт `check_output.py`. Пример вывода:
- Всего комментариев: 14412
- С матом: 2992 (20.76%)
- Без мата: 11420 (79.24%)

### Скрипт аналитики `check_output.py`
Этот скрипт предназначен для проверки результатов обработки и получения статистики. Он выполняет следующие действия:
1. Читает выходной файл `obscene_with_flag.csv`.
2. Подсчитывает количество строк с метками `yes` и `no`.
3. Выводит общее количество комментариев, количество с матом и без, а также процентное соотношение.
4. Выводит несколько случайных строк для визуальной проверки корректности работы.

Для запуска выполните:
```bash
py check_output.py
```
или
```bash
python check_output.py
```

Скрипт использует только стандартные библиотеки Python (csv, random).

## Сравнение трех моделей детекции мата

В проекте представлены три различные модели детекции нецензурных выражений:

1. **GPT5** (`add_column_modify.py`) - использует расширенный список регулярных выражений с нормализацией текста
2. **DeepSeek** (`add_column.py`) - использует базовый список регулярных выражений без нормализации
3. **AI Agent** (`add_column_modify with_AI.py`) - комбинирует регулярные выражения с ML моделью `rubert-tiny-toxicity`

### Результаты сравнения (на общих 622 комментариях)

#### Статистика по моделям:
- **GPT5**: 38.88% комментариев с матом (5604 из 14412)
- **DeepSeek**: 20.65% комментариев с матом (2923 из 14155)  
- **AI Agent**: 18.22% комментариев с матом (2563 из 14066)

#### Согласованность моделей:
- **GPT5 vs DeepSeek**: 72.51% совпадений
- **GPT5 vs AI Agent**: 76.37% совпадений
- **DeepSeek vs AI Agent**: 86.82% совпадений

#### Консенсус всех трех моделей:
- Полное согласие (все yes): 17.20%
- Полное согласие (все no): 50.64%
- Разногласия (хотя бы одна модель отличается): 32.15%

### Выводы:
1. **GPT5** является наиболее "строгой" моделью, обнаруживая мат в 38.88% комментариев
2. **AI Agent** - наиболее "консервативная" модель (18.22%)
3. **DeepSeek** и **AI Agent** показывают наибольшую согласованность (86.82%)
4. Во всех трех моделях наблюдается значительное расхождение (32.15% комментариев имеют разную классификацию)
5. GPT5 классифицирует почти в 2 раза больше комментариев как содержащих мат по сравнению с другими моделями

### Примеры разногласий:
Из отчета видно, что GPT5 часто помечает как "мат" комментарии, которые другие модели считают "чистыми". Например:
- "Так, чисто поржать, ибо пьян.." - GPT5: yes, другие: no
- "Барином хорошо было быть, в 17-ом веке..." - GPT5: yes, другие: no

### Расширенный скрипт `check_output.py`
Обновленный скрипт теперь поддерживает:
- Сравнение результатов трех моделей
- Вычисление метрик согласованности
- Генерацию отчета в файл `comparison_report.txt`
- Создание детального CSV файла `model_comparison.csv` с классификациями всех моделей

Для запуска сравнения выполните:
```bash
py check_output.py
```

## Примечания
- Исходный файл не изменяется.
- Для обработки других файлов необходимо изменить переменные `input_file` и `output_file` в соответствующих скриптах.
- При необходимости можно расширить список регулярных выражений в переменной `obscene_patterns` внутри скриптов.
- Для работы расширенного скрипта `check_output.py` требуются библиотеки: `pandas`, `scikit-learn`, `matplotlib`, `seaborn`.
