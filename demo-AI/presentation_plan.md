## Проект 1: Система модерации контента (Project Obscene)

### Описание

Автоматизированная система анализа пользовательских комментариев на наличие нецензурных выражений с использованием трех AI-моделей.

### Ключевые метрики

- **Обработано комментариев:** 14 000+
- **Точность детекции:** до 86.8% согласованность между моделями
- **Скорость обработки:** автоматическая классификация в реальном времени

### Преимущества для бизнеса

| Преимущество | Описание |
|-------------|----------|
| Масштабируемость | Обработка неограниченного объема комментариев |
| Автоматизация | Снижение потребности в ручной модерации до 80% |
| Многоуровневая проверка | Три модели обеспечивают перекрестную валидацию |
| Снижение рисков | Защита бренда от негативного контента |

### Структура презентации

1. **Проблема** — необходимость модерации User Generated Content
2. **Решение** — AI-система автоматической детекции
3. **Демонстрация** — примеры работы системы
4. **Метрики** — точность, скорость, экономия ресурсов
5. **ROI** — потенциальная экономия на ручной модерации

## Проект 2: Система управления заказами (Temporal OMS)

### Описание

Демонстрационная система управления заказами на базе платформы Temporal, реализующая надежную оркестрацию бизнес-процессов с гарантией выполнения.

### Ключевые возможности

- **Надежность** — workflow продолжает выполнение после сбоев
- **Отслеживаемость** — полная история выполнения каждого заказа
- **Гибкость** — легкое добавление новых этапов обработки
- **Масштабируемость** — горизонтальное масштабирование workers

### Бизнес-процесс

```
Создание заказа → Проверка запасов → Обработка платежа 
    → Подготовка → Отправка → Доставка
```

### Преимущества для бизнеса

| Преимущество | Бизнес-ценность |
|-------------|-----------------|
| Отказоустойчивость | Гарантированное выполнение заказов даже при сбоях |
| Прозрачность | Полный контроль над процессом обработки |
| Скорость внедрения | Быстрое добавление новых бизнес-правил |
| Интеграция | Легкое подключение к существующим системам |

### Структура презентации

1. **Концепция** — что такое workflow-оркестрация
2. **Архитектура** — компоненты системы (Client, Workflow, Activity, Worker, Server)
3. **Демонстрация** — визуализация процесса обработки заказа
4. **Сравнение** — традиционный подход vs Temporal
5. **Применимость** — для каких бизнес-процессов подходит

## Рекомендации по подаче материала

### Для руководства важно

- Фокус на бизнес-ценность, не на технические детали
- Конкретные метрики и ROI
- Риски и их митигация
- Roadmap внедрения

### Визуальные материалы

- Инфографика: сравнение подходов к модерации
- Диаграмма: архитектура Temporal
- Графики: метрики точности моделей
- Примеры: скриншоты работы систем

### Ответы на возможные вопросы

1. **Сколько стоит внедрение?** — Зависит от масштаба, но AI-подход дешевле ручной модерации
2. **Какие риски?** — Точность 86-87% требует human-in-the-loop для спорных случаев
3. **Как интегрировать?** — Оба решения могут быть интегрированы через API
4. **Кто поддерживает?** — Код открыт, возможно расширение командой разработки
