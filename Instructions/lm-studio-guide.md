# LM Studio — Локальные LLM модели

## Установка LM Studio

1. Скачать с официального сайта: https://lmstudio.ai
2. Установить как обычную программу
3. После установки данные хранятся в: `~/.lmstudio/`

---

## Скачивание модели

### Способ 1 — Через поиск в LM Studio

1. Открыть LM Studio
2. Перейти на вкладку **Search** (поиск моделей)
3. Ввести название модели, например: `Qwen3.5-9B`
4. Выбрать нужный файл (GGUF) и нажать **Download**

### Способ 2 — Скачать вручную с Hugging Face

1. Зайти на страницу модели на Hugging Face (например, https://huggingface.co/models)
2. Перейти на вкладку **Files**
3. Скачать нужный GGUF-файл (например, `Q4_K_M` — оптимальный баланс качества/размера)
4. Поместить в папку: `~/.lmstudio/models/<организация>/`

**Структура папок:**
```
~/.lmstudio/models/
└── org-name/
    ├── Model-Name-Q4_K_M.gguf
    └── mmproj-Model-Name-BF16.gguf (опционально, для изображений)
```

### Какой квант выбрать

| Квант | Размер | Когда использовать |
|-------|--------|-------------------|
| Q4_K_M | ~5.6 GB | **Рекомендуется** — лучший баланс качества/памяти |
| Q6_K | ~7.4 GB | Если есть >8GB VRAM |
| Q8_0 | ~9.5 GB | Если есть >12GB VRAM |
| BF16 | ~17 GB | Полное качество, только если много памяти |

## Запуск модели

1. Открыть LM Studio
2. Перейти на вкладку **Chat**
3. В выпадающем списке сверху выбрать нужную модель
4. Нажать **Load Model**
5. Дождаться загрузки (обычно 10-30 секунд)

### Рекомендуемые настройки при загрузке

| Параметр | Thinking mode | Non-thinking mode |
|----------|--------------|-------------------|
| Temperature | 0.6 | 0.7 |
| Top P | 0.95 | 0.8 |
| Top K | 20 | 20 |
| Context Length | 131072 (128K) | 131072 (128K) |

> **Важно:** Context Length не меньше 128K — иначе модель теряет способность к мышлению.

## Подключение к Kilo Code / другим приложениям

LM Studio умеет запускать локальный сервер, совместимый с OpenAI API.

### Включение сервера

1. В правой боковой панели LM Studio включить переключатель **Local Service** (или **Start Server**)
2. Сервер будет доступен по адресу: `http://localhost:1234/v1`

### Настройка в Kilo Code

```json
{
  "providers": {
    "lm-studio": {
      "type": "openai",
      "apiUrl": "http://localhost:1234/v1",
      "apiKey": "not-needed"
    }
  }
}
```

### Проверка через curl

```powershell
curl.exe -X POST http://localhost:1234/v1/chat/completions `
  -H "Content-Type: application/json" `
  -d '{
    "model": "your-model-name",
    "messages": [{"role": "user", "content": "Привет!"}]
  }'
```

## Частые проблемы

**Модель не загружается / ошибка "Unsupported architecture"**
→ Обновить LM Studio до последней версии.

**Модель грузится только в CPU (очень медленно)**
→ В настройках загрузки модели проверить `GPU Offload` — выставить максимальное значение.

**Не хватает оперативной памяти**
→ Использовать модель с меньшим квантом (Q4_K_M вместо Q8_0) или модель меньшего размера (4B вместо 9B).

**Сервер не запускается**
→ Проверить, не занят ли порт 1234 другой программой. Можно сменить порт в настройках LM Studio.
