# Temporal Demo - Система управления заказами (OMS)

Это демо-проект для изучения Temporal на примере системы управления заказами (Order Management System).

## Структура проекта

```
temporal/
├── README.md                 # Документация проекта
├── package.json              # Зависимости проекта
├── src/
│   ├── workflows/            # Workflow определения
│   │   └── order-workflow.ts # Workflow процесса заказа
│   ├── activities/           # Activity функции
│   │   └── order-activities.ts # Бизнес-процессы заказа
│   ├── client/               # Клиент для запуска workflow
│   │   └── order-client.ts   # Клиент заказа
│   └── worker/               # Worker для выполнения workflow
│       └── order-worker.ts   # Worker заказа
└── docker-compose.yml        # Docker для Temporal сервера
```

## Описание демо

Демо показывает процесс обработки заказа в системе OMS с использованием Temporal:

1. **Создание заказа** - клиент создает новый заказ
2. **Проверка запасов** - проверка наличия товаров на складе
3. **Проверка платежа** - валидация платежной информации
4. **Подготовка к отправке** - сборка и упаковка заказа
5. **Отправка заказа** - передача в службу доставки
6. **Уведомление клиента** - отправка статуса заказа

## Установка и запуск

1. Установите зависимости:
```bash
npm install
```

2. Запустите Temporal сервер:
```bash
docker-compose up -d
```

3. Запустите worker:
```bash
npm run worker
```

4. Запустите клиент (в отдельном терминале):
```bash
npm run client
```

## Temporal концепции

- **Workflow**: Долгоживущий процесс обработки заказа
- **Activity**: Отдельные бизнес-операции (проверка запасов, платежа и т.д.)
- **Worker**: Процесс, который выполняет workflow и activity
- **Client**: Запускает workflow и запрашивает статус
