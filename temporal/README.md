## 🎯 О проекте

Этот проект демонстрирует ключевые концепции Temporal на практическом примере обработки заказов в системе OMS. Вы увидите как Temporal обеспечивает надежное выполнение долгоживущих бизнес-процессов.

### Ключевые возможности Temporal:

- **Надежность** - Workflow продолжают выполнение после сбоев
- **Отслеживаемость** - Полная история выполнения каждого workflow
- **Гибкость** - Легко изменять бизнес-логику и добавлять новые этапы
- **Масштабируемость** - Горизонтальное масштабирование workers

## 🏗️ Архитектура Temporal

### Компоненты системы:

1. **👤 Клиент** - Запускает workflow и взаимодействует с ними
2. **🔄 Workflow** - Долгоживущий бизнес-процесс (обработка заказа)
3. **🔧 Activity** - Отдельные бизнес-операции (проверка запасов, платеж и т.д.)
4. **⚙️ Worker** - Выполняет workflow и activity
5. **🏢 Temporal Server** - Оркестрирует выполнение workflow

### Workflow процесса заказа:

```
📝 СОЗДАНИЕ → 📦 ПРОВЕРКА ЗАПАСОВ → 💳 ОБРАБОТКА ПЛАТЕЖА 
    ↓
🎁 ПОДГОТОВКА → 🚚 ОТПРАВКА → ✅ ДОСТАВКА
```

### Activity функции:

- `checkInventory()` - Проверка наличия товаров
- `processPayment()` - Обработка платежа
- `prepareForShipment()` - Подготовка к отправке
- `shipOrder()` - Отправка заказа
- `notifyCustomer()` - Уведомление клиента
- `cancelOrder()` - Отмена заказа

## 💡 Концепции Temporal в демо

### Workflow
- **Долгоживущий процесс** обработки заказа
- **Сохраняет состояние** между перезапусками
- **Оркестрирует** выполнение activity функций
- **Обрабатывает сигналы** (например, отмену заказа)

### Activity
- **Отдельные бизнес-операции**
- **Могут содержать** любой код (API вызовы, работа с БД)
- **Имеют таймауты** и политики повторных попыток
- **Выполняются изолированно** от workflow

### Преимущества Temporal:
- **Надежность** - Workflow продолжают выполнение после сбоев серверов
- **Отслеживаемость** - Полная история выполнения в Temporal UI
- **Гибкость** - Легко добавлять новые этапы в бизнес-процесс
- **Масштабируемость** - Горизонтальное масштабирование workers

## 🐛 Устранение проблем

### Распространенные проблемы:

1. **Node.js не установлен** - Используйте PowerShell или браузерную версию
2. **Docker не запущен** - Запустите Docker Desktop
3. **Порты заняты** - Измените порты в `docker-compose.yml`

### Решения:

```bash
# Проверьте что сервисы запущены
docker-compose ps

# Перезапустите Temporal сервер
docker-compose restart temporal

# Проверьте установку Node.js
node --version
npm --version
```
