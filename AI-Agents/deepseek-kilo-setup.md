# Инструкция: Как включить и проверить модель DeepSeek Reasoner в Kilo Code

## Предварительные требования

1. **Аккаунт DeepSeek**: Зарегистрируйтесь на [platform.deepseek.com](https://platform.deepseek.com/)
2. **API ключ**: Получите API ключ в личном кабинете DeepSeek
3. **Установленный Kilo Code**: Расширение Kilo Code должно быть установлено в VS Code

## Шаг 1: Получение API ключа DeepSeek

1. Перейдите на [platform.deepseek.com](https://platform.deepseek.com/)
2. Войдите в свой аккаунт или создайте новый
3. В личном кабинете перейдите в раздел "API Keys"
4. Нажмите "Create new API key"
5. Скопируйте ключ (он показывается только один раз!)

## Шаг 2: Настройка Kilo Code для работы с DeepSeek Reasoner

### Вариант A: Через интерфейс Kilo Code (рекомендуется)

1. **Откройте панель Kilo Code** в VS Code
   - Нажмите на иконку Kilo Code в боковой панели
   - Или используйте команду `Ctrl+Shift+P` → "Kilo Code: Open"

2. **Перейдите в настройки**:
   - Нажмите на значок ⚙️ (шестеренка) в панели Kilo Code
   - Или используйте команду `Ctrl+Shift+P` → "Kilo Code: Settings"

3. **Настройте провайдера API**:
   - В разделе **"API Provider"** выберите **"DeepSeek"**
   - В поле **"DeepSeek API Key"** вставьте ваш API ключ
   - В выпадающем списке **"Model"** выберите **`deepseek-reasoner`**

4. **Дополнительные настройки** (опционально):
   - **Temperature**: 0.7 (рекомендуется для баланса креативности и точности)
   - **Max Tokens**: 4096 (или больше для сложных задач)
   - **Context Window**: 128K (автоматически настраивается)

### Вариант B: Через конфигурационные файлы

1. **Найдите файл настроек Kilo Code**:
   ```
   C:\Users\user\AppData\Roaming\Code\User\globalStorage\kilocode.kilo-code\settings\
   ```

2. **Создайте или отредактируйте файл конфигурации**:
   - Файл может называться `config.json` или `settings.json`
   - Добавьте следующую конфигурацию:

   ```json
   {
     "aiProvider": "deepseek",
     "deepseekApiKey": "ВАШ_API_КЛЮЧ",
     "model": "deepseek-reasoner",
     "temperature": 0.7,
     "maxTokens": 4096
   }
   ```

## Шаг 3: Проверка подключения

### Тест 1: Простой запрос на логическое мышление

Отправьте следующий запрос в Kilo Code:

```
Реши математическую задачу: У Маши было 5 яблок. Она отдала 2 яблока Пете и купила еще 3 яблока. Сколько яблок стало у Маши? Покажи пошаговое решение.
```

**Ожидаемый результат от DeepSeek Reasoner**:
- Модель покажет цепочку рассуждений (Chain of Thought)
- Пошаговое решение с объяснениями
- Итоговый ответ: 6 яблок

### Тест 2: Сложная логическая задача

```
Реши логическую задачу: В комнате находятся 3 выключателя, каждый подключен к одной из трех лампочек в соседней комнате. Ты можешь включить выключатели, но войти в комнату с лампочками можно только один раз. Как определить, какой выключатель к какой лампочке подключен?
```

**Ожидаемый результат**:
- Детальное пошаговое рассуждение
- Логическое обоснование решения
- Четкий план действий

### Тест 3: Программирование с отладкой

```
Напиши функцию на Python, которая проверяет, является ли строка палиндромом. Затем найди и исправь ошибку в следующем коде:

def is_palindrome(s):
    return s == s.reverse()
```

**Ожидаемый результат**:
- Объяснение, почему код не работает
- Правильная реализация функции
- Примеры использования

## Шаг 4: Верификация работы модели

### Признаки успешной работы DeepSeek Reasoner:

1. **Цепочка рассуждений (Chain of Thought)**:
   - Модель показывает промежуточные шаги мышления
   - Ответ содержит разделы "Рассуждение" и "Итоговый ответ"

2. **Более детальные ответы**:
   - Ответы длиннее и содержательнее
   - Больше объяснений и обоснований

3. **Лучшая точность в сложных задачах**:
   - Меньше ошибок в математических вычислениях
   - Более точные логические выводы

### Проверка текущей модели:

Чтобы проверить, какая модель сейчас активна, отправьте запрос:

```
Какая модель AI сейчас используется? Назови свою модель и режим работы.
```

DeepSeek Reasoner должен ответить примерно так:
```
Я работаю на модели DeepSeek Reasoner (deepseek-reasoner) в режиме цепочки рассуждений (Chain of Thought).
```

## Шаг 5: Переключение между моделями

### Чтобы вернуться к DeepSeek Chat:

1. Откройте настройки Kilo Code
2. В выпадающем списке **"Model"** выберите **`deepseek-chat`**
3. Сохраните изменения

### Различия в работе:

| Аспект | DeepSeek Chat | DeepSeek Reasoner |
|--------|---------------|-------------------|
| **Скорость** | Быстрее | Медленнее (из-за CoT) |
| **Детализация** | Краткие ответы | Подробные с рассуждениями |
| **Идеально для** | Повседневные задачи, чат | Сложный анализ, математика, отладка |
| **Стоимость** | Ниже | Выше (больше токенов) |

## Шаг 6: Устранение неполадок

### Проблема 1: "Invalid API key"
- **Решение**: Проверьте правильность API ключа
- **Действие**: Получите новый ключ на platform.deepseek.com

### Проблема 2: Модель не показывает цепочку рассуждений
- **Решение**: Включите отображение thinking blocks
- **Действие**: В настройках Kilo Code найдите опцию "Always Show Thinking Blocks"

### Проблема 3: Медленные ответы
- **Причина**: DeepSeek Reasoner требует больше времени на обработку
- **Решение**: Увеличьте timeout в настройках или используйте для сложных задач

### Проблема 4: Высокая стоимость
- **Причина**: CoT использует больше токенов
- **Решение**: Используйте DeepSeek Reasoner только для задач, требующих глубокого анализа

3. **Комбинируйте модели**: Используйте Reasoner для сложных задач, Chat для простых

## Заключение

DeepSeek Reasoner предоставляет мощные возможности для сложного логического мышления и анализа. Для его использования в Kilo Code необходимо:

1. Получить API ключ DeepSeek
2. Настроить Kilo Code на использование `deepseek-reasoner`
3. Проверить работу с тестовыми запросами
4. Оптимизировать настройки под конкретные задачи

Модель особенно полезна для:
- Решения математических задач
- Отладки кода и анализа алгоритмов
- Сложного логического анализа
- Научных исследований и вычислений

Для повседневных задач и быстрых ответов рекомендуется использовать DeepSeek Chat, переключаясь на Reasoner только когда требуется глубокая аналитика.
